陣列去重：深度效能分析
Set O(n) filter+indexOf O(n²) ⇵ 黃金交叉
黃金交叉：n ≈ ?  ·  小於此值 Linear 更快（Hash 建表開銷 > 掃描成本）；大於此值 Set 更快（O(1) 查找）
Set 在百萬規模仍穩定；filter+indexOf 在此規模需要數十分鐘
filter+indexOf 在 n=100 萬理論耗時（O(n²) 推算）： ~17 分鐘
